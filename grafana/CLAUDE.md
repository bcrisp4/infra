# grafana/ — git-synced dashboards

Dashboards here sync to the "Infrastructure" folder (see root `CLAUDE.md` for the Git Sync setup, datasource UID, and the unmanaged-folder gotcha). Conventions for authoring dashboard JSON:

- Tabbed / dynamic-layout dashboards (tabs, nested rows) require schema `dashboard.grafana.app/v2`, NOT classic schemaVersion 39. Commit as a wrapped manifest file: `{apiVersion:"dashboard.grafana.app/v2", kind:"Dashboard", metadata:{name:<uid>}, spec:{...}}`. Classic bare JSON (e.g. `bns-overview.json`) still works (parsed as v0alpha1); both formats coexist here. Layout nests: `spec.layout` TabsLayout → `tabs[]` → RowsLayout → `rows[]` → GridLayout → items referencing `spec.elements["panel-N"]`.
- v2 wraps EVERYTHING as `{kind, spec}` — including transformations: `{kind:<id>, spec:{id:<id>, options:{...}}}`. Omitting `kind` defaults to `""` and crashes the whole dashboard at render (`"" not found in [reduce,...,merge,...]`). Model panel/query/variable/viz JSON off an existing dashboard's v2 form (`GET /apis/dashboard.grafana.app/v2/namespaces/default/dashboards/<uid>`); full schema at `/openapi/v3/apis/dashboard.grafana.app/v2beta1`.
- Validate a v2 spec by POST to `/apis/dashboard.grafana.app/v2/namespaces/default/dashboards` (201 = schema-valid) — but POST does NOT catch render-time errors. After deploy, `get_panel_image` (MCP) renders panels/whole dashboards to PNG for a programmatic visual check (image-renderer is deployed on rpi5); pass `var-job`/`var-instance` and `${datasource}` resolves to the default. Variable `hide`: `0`/`dontHide`, `1`/`hideLabel`, `2`/`hideVariable`.
- Grafana MCP token can create/POST dashboards but lacks `dashboards:delete` and GET-on-individual (403) — delete scratch/test dashboards manually in the UI.
- For many-panel dashboards, generating the JSON with a throwaway script (not committed) and `cp`-ing it into `grafana/` beats hand-editing or piping large bodies through the MCP.
